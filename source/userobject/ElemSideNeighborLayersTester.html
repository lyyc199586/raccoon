ElemSideNeighborLayersTester

User object to calculate ghosted elements on a single processor or the union across all processors.

Overview

A user object to test whether or not ghost elements are correctly reserved. element_side_neighbor_layers is used to specify the number of layers of ghosting elements that will be preserved via either a geometric or an algebraic relationship manager.

Input Parameters

  • element_side_neighbor_layersNumber of layers to ghost

    C++ Type:unsigned short

    Options:

    Description:Number of layers to ghost

Required Parameters

  • execute_onTIMESTEP_BEGINThe list of flag(s) indicating when this object should be executed, the available options include NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM.

    Default:TIMESTEP_BEGIN

    C++ Type:ExecFlagEnum

    Options:NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM

    Description:The list of flag(s) indicating when this object should be executed, the available options include NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM.

  • prop_getter_suffixAn optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

    C++ Type:MaterialPropertyName

    Options:

    Description:An optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

  • rank4294967295The rank for which the ghosted elements are recorded (Default: ALL)

    Default:4294967295

    C++ Type:unsigned int

    Options:

    Description:The rank for which the ghosted elements are recorded (Default: ALL)
