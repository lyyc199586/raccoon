ActivateElementsByPath

This user object uses the function path as the metric to activate (add) an element by moving the element from an "inactive" subdomain to the "active" subdomain. It uses the user provided points with components defined by the functions specified by the parameters function_x, function_y, and function_z in the input. An element is activated at time if this element is close (distance < activate_distance) to the point .

Input Parameters

  • active_subdomain_idThe active subdomain ID.

    C++ Type:unsigned short

    Options:

    Description:The active subdomain ID.

  • expand_boundary_nameThe expanded boundary name.

    C++ Type:std::vector<BoundaryName>

    Options:

    Description:The expanded boundary name.

Required Parameters

  • activate_distance0.0001The maximum distance of the activated element to the point on the path.

    Default:0.0001

    C++ Type:double

    Options:

    Description:The maximum distance of the activated element to the point on the path.

  • blockThe list of blocks (ids or names) that this object will be applied

    C++ Type:std::vector<SubdomainName>

    Options:

    Description:The list of blocks (ids or names) that this object will be applied

  • execute_onTIMESTEP_ENDThe list of flag(s) indicating when this object should be executed, the available options include NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM.

    Default:TIMESTEP_END

    C++ Type:ExecFlagEnum

    Options:NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM

    Description:The list of flag(s) indicating when this object should be executed, the available options include NONE, INITIAL, LINEAR, NONLINEAR, TIMESTEP_END, TIMESTEP_BEGIN, FINAL, CUSTOM.

  • function_x0The x component of the heating spot travel path

    Default:0

    C++ Type:FunctionName

    Options:

    Description:The x component of the heating spot travel path

  • function_y0The y component of the heating spot travel path

    Default:0

    C++ Type:FunctionName

    Options:

    Description:The y component of the heating spot travel path

  • function_z0The z component of the heating spot travel path

    Default:0

    C++ Type:FunctionName

    Options:

    Description:The z component of the heating spot travel path

  • inactive_subdomain_id65535The inactivate subdomain ID, i.e., the subdomain that you want to keep the same.

    Default:65535

    C++ Type:unsigned short

    Options:

    Description:The inactivate subdomain ID, i.e., the subdomain that you want to keep the same.

  • prop_getter_suffixAn optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

    C++ Type:MaterialPropertyName

    Options:

    Description:An optional suffix parameter that can be appended to any attempt to retrieve/get material properties. The suffix will be prepended with a '_' character.

    Description:In the case where this UserObject is depended upon by an initial condition, allow it to be executed twice during the initial setup (once before the IC and again after mesh adaptivity (if applicable).

    Description:Forces the UserObject to be executed in POSTAUX

    C++ Type:bool

    Options:

    Description:Forces the UserObject to be executed in PREAUX

    Description:Forces the UserObject to be executed in PREIC during initial setup

    Description:Determines whether this object is calculated using an implicit or explicit form
